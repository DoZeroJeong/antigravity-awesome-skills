# SQL 최적화 패턴

체계적인 최적화, 올바른 인덱싱 및 쿼리 플랜 분석을 통해 느린 데이터베이스 쿼리를 매우 빠른 작업으로 탈바꿈시킵니다.

## 이 스킬을 사용할 때

- 실행이 느린 쿼리를 디버깅할 때
- 성능이 뛰어난 데이터베이스 스키마를 설계할 때
- 애플리케이션 응답 시간을 최적화할 때
- 데이터베이스 부하 및 비용을 줄일 때
- 커지는 데이터셋에 대비해 확장성을 개선할 때
- EXPLAIN 쿼리 실행 계획을 분석할 때
- 효율적인 인덱스를 구현할 때
- N+1 쿼리 문제를 해결할 때

## 이 스킬의 사용을 지양할 때

- 작업이 SQL 최적화 패턴과 무관할 때
- 이 범위를 벗어난 다른 도메인이나 도구가 필요할 때
