# Python Testing Patterns

pytest, fixtures, mocking, 매개변수화(parameterization) 및 테스트 주도 개발(TDD) 관행을 사용하여 Python에서 견고한 테스트 전략을 구현하기 위한 포괄적인 가이드입니다.

## 이 스킬의 사용 시점

- Python 코드에 대한 단위 테스트(Unit tests)를 작성할 때
- 테스트 스위트 및 테스트 인프라를 설정할 때
- 테스트 주도 개발(TDD)을 구현할 때
- API 및 서비스에 대한 통합 테스트(Integration tests)를 생성할 때
- 외부 의존성 및 서비스를 Mocking할 때
- 비동기 코드 및 동시성 작업을 테스트할 때
- CI/CD에서 지속적인 테스트를 설정할 때
- 속성 기반 테스트(Property-based testing)를 구현할 때
- 데이터베이스 작업을 테스트할 때
- 실패하는 테스트를 디버깅할 때

## 이 스킬을 사용하지 말아야 할 시점

- 작업이 Python 테스트 패턴과 관련이 없을 때
- 이 범위 밖의 다른 도메인이나 도구가 필요할 때
