# Verify Skill

## 개요 (Overview)

이 스킬은 특정 스킬이 올바르게 생성되었는지 검증하는 절차를 가이드합니다. 스킬 디렉토리 구조, 필수 파일 존재 여부, 그리고 `SKILL.md` 파일의 내용이 표준을 따르는지 확인하여 스킬의 품질을 보장합니다.

## 사용 시점 (When to use)

- 새로운 스킬을 생성한 직후
- 기존 스킬을 수정하거나 리팩토링한 후
- 스킬이 예상대로 동작하지 않아 구조적 문제를 확인하고 싶을 때

## 지침 (Instructions)

검증할 스킬의 이름(이하 `<skill-name>`)을 확인하고 다음 단계를 수행하세요.

1. **디렉토리 존재 확인:**
   - `.agent/skills/<skill-name>/` 디렉토리가 존재하는지 확인합니다.
   - 도구: `list_dir`

2. **필수 파일 확인:**
   - 해당 디렉토리 안에 `SKILL.md` 파일이 존재하는지 확인합니다.
   - 도구: `list_dir`

3. **SKILL.md 구조 및 내용 검증:**
   - `SKILL.md` 파일의 내용을 읽습니다.
   - 도구: `view_file`
   - **Checklist:**
     - [ ] **YAML Frontmatter:** 파일 최상단에 `name`과 `description`이 포함된 YAML 블록이 있는가?
     - [ ] **제목:** `# <Skill Title>` 형식이 있는가?
     - [ ] **개요 (Overview):** 스킬의 목적과 기능을 설명하는 섹션이 있는가?
     - [ ] **사용 시점 (When to use):** 언제 이 스킬을 사용해야 하는지 명시되어 있는가?
     - [ ] **지침 (Instructions):** 단계별 수행 절차가 명확하게 기술되어 있는가?

4. **가독성 및 명확성 평가:**
   - 내용이 이해하기 쉽고, 에이전트가 수행하기에 모호함이 없는지 판단합니다.
   - 예시가 포함되어 있다면 더욱 좋습니다.

## 검증 결과 (Verification Result)

- 모든 항목을 통과했다면: "✅ 스킬 `<skill-name>` 검증 완료: 모든 표준을 준수합니다."
- 부족한 점이 있다면: "❌ 스킬 `<skill-name>` 검증 실패: [부족한 항목] 수정이 필요합니다." 라고 보고하고 수정을 제안하세요.
